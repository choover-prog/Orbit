# Quiet Orbit Composition

## Selected direction

The refined main route uses the Presence-centered direction.

The vertical relationship is:

1. Orbit Presence
2. personal greeting and semantic state
3. one focal concern
4. one primary conversational action
5. optional evidence or context
6. voice and keyboard input

Maya is represented through the greeting rather than a permanent profile card. Orbit remains the active identity, and the focal concern stays within the same centered reading path.

## Hierarchy

- State and domain labels are small, restrained, and specific.
- The focal headline is editorial but capped below display-hero scale.
- One supporting sentence explains the consequence.
- `Talk it through` is the primary action.
- `Why this matters` progressively reveals the two source facts.
- Additional eligible concerns are acknowledged without being listed.

There are no decorative orbit lines, side rails, dashboard cards, domain tabs, or arbitrary horizontal rules in the daily surface.

## Conversation and action continuity

Conversation and approval screens keep a medium Presence near the active content. Speaking uses the expressive state; proposal review uses noticing. This prevents Orbit from disappearing once the user enters the actual decision flow.

## Navigation

History, Connections, Settings, and the development-only Presence Lab remain keyboard accessible inside a compact native menu. The daily concern no longer competes with a persistent row of utility links.

## Responsive behavior

The same semantic order is preserved on desktop and mobile. Compact screens reduce headline scale, retain the full Presence, keep the input reachable, and avoid horizontal overflow. The user is never converted into a side card at wider breakpoints.
