# Presence Lab Refinement

The Presence Lab is a personality and motion studio, not a settings screen.

## First viewport

The first viewport now gives most of its area to one large live Presence. It pairs the selected variant, current state, personality sentence, semantic state explanation, replay control, and context/comparison mode switch. Controls follow the hero instead of leading the experience.

## Controls

Variant, state, intensity, speed, simulated audio, size, theme, animation, and reduced-motion controls remain available in a quiet lower disclosure panel. The selected variant still persists locally. No backend or account persistence was added.

## Comparison mode

All nine variants appear at the same large scale and in the same selected state. The grid auto-fits available width instead of forcing every variant into one cramped row. Each preview includes a short personality description rather than only a loader-like label.

## Liquid concept import

The liked concept directions have been moved into the live lab as:

- `Mercury Loop`, based on the open liquid-metal ring concept.
- `Elastic Halo`, based on the translucent ribbon halo concept.
- `Morph Core`, based on the liquid flubber concept with a center pulse and content notification bead.

## Context mode

Context mode uses three full-size product frames:

1. Quiet resting screen with idle Presence.
2. Single-attention screen with attention Presence.
3. Focus conversation screen with speaking Presence.

These frames evaluate the selected identity across realistic product moments rather than showing three tiny cards with the same state.

## Sequence mode

Replay demonstrates the canonical semantic sequence and remains interruptible by selecting another state. The control is disabled while the bounded sequence is playing and returns to an idle-ready state.
