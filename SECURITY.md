# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository when available. Include reproduction steps, affected components, impact, and any suggested mitigation.

## Sensitive data

Never include credentials, tokens, private messages, health information, contact details, calendar data, location history, or other real personal records in reports, fixtures, screenshots, logs, or commits. Replace sensitive values with fictional examples.

## Supported versions

Orbit is currently pre-release. Security fixes apply to the latest `main` branch until a versioned support policy is published.
